# 인증 VS 인가
# 인증(Authentication)과 인가(Authorization)의 차이

## 인증 (Authentication)
사용자가 **누구인지 확인하는 과정**

즉, 로그인 과정을 통해 사용자의 신원을 검증하는 것이다.

### 예시
- 아이디 / 비밀번호 로그인
- 지문 인식
- OTP 인증
- JWT 토큰 검증


---

## 인가 (Authorization)
인증된 사용자가 **무엇을 할 수 있는지 확인하는 과정**

즉, 해당 사용자가 특정 자원이나 기능에 접근할 권한이 있는지 검사하는 것이다.

### 예시
- 관리자만 관리자 페이지 접근 가능
- 일반 사용자는 게시글 조회만 가능
- 작성자만 게시글 수정 / 삭제 가능

---

## 쉽게 비유하면

### 학교
- **인증**: 학생증 검사 → 이 학교 학생이 맞는지 확인
- **인가**: 교무실 출입 가능 여부 확인 → 들어갈 권한이 있는지 확인

### 웹사이트
- **인증**: 로그인
- **인가**: 로그인 후 관리자 페이지 접근 권한 확인

---

## 순서
1. **인증(Authentication)** 진행  
2. **인가(Authorization)** 진행  


---

## 정리
| 구분 | 인증(Authentication) | 인가(Authorization) |
|-----|---------------------|---------------------|
| 의미 | 사용자가 누구인지 확인 | 무엇을 할 수 있는지 확인 |
| 질문 | 너 누구야? | 너 이거 해도 돼? |
| 예시 | 로그인, 지문, OTP | 관리자 권한, 수정/삭제 권한 |
